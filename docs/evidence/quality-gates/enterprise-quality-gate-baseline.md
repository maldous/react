# Enterprise quality gate baseline evidence

## Summary

Added external quality, security, and documentation tooling before the first vertical slice (ADR-ACT-0008). The gates complement `validate-source-imports` as the authoritative ADR import-boundary checker and do not duplicate its rules.

## Hard gates

These checks must pass on every PR and push to main:

1. **format:check** ŌĆö Prettier formatting check; covers `.md`, `.json`, `.yml`, `.mjs`, `.ts`, `.tsx`. Deterministic: same input always produces same output.
2. **lint:md** ŌĆö markdownlint-cli2; enforces markdown structure in all authored docs. Disabled: MD013 (line length), MD024 (duplicate headings), MD033 (inline HTML).
3. **lint** ŌĆö ESLint flat config; two-bucket pattern:
   - Bucket 1: `tools/architecture/**/*.mjs` ŌĆö Node.js globals, `@eslint/js` recommended
   - Bucket 2: `packages/**/*.{ts,tsx}`, `apps/**/*.{ts,tsx}` ŌĆö TypeScript eslint recommended, browser globals
   - Rules: `no-unused-vars` (underscore exception), `no-unreachable`, `no-duplicate-imports`
   - Does NOT encode ADR import boundaries (validate-source-imports owns that)
4. **audit:deps** ŌĆö `npm audit --audit-level=high`; fails on high/critical CVEs.
5. **audit:osv** ŌĆö `osv-scanner scan --recursive .`; scans all lock files against the OSV database.
6. **secrets:scan** ŌĆö `gitleaks/gitleaks-action@v2` in CI; detects committed secrets.

## Report-only gates

These run in CI but do not fail the build. Expected to produce output for skeleton packages.

- **knip** ŌĆö Reports unused files, exports, and dependencies. Skeleton packages will flag `packageName` exports as unused ŌĆö this is expected state.
- **depcruise** ŌĆö Dependency graph smoke tests: no-circular (error), product must not import architecture tooling (error), no imports from reports/ or docs/ (warn).

## Ignored paths

All tools exclude:

- `**/node_modules/**`
- `reports/` (generated, gitignored)
- `tools/architecture/**/tests/fixtures/` (intentionally broken code)

Prettier additionally excludes:

- `docs/evidence/` (byte-stable committed governance evidence)
- `**/README.md` in apps/packages/tools (generated by generate-package-readmes)
- `**/package-lock.json`

markdownlint additionally excludes:

- `.remember/**` (auto-generated session memory, gitignored via `.remember/.gitignore`)

## Bugs fixed during implementation

- `validate-lifecycle-evidence` used `Ajv` (draft-07) instead of `Ajv2020` (draft-2020-12). Fixed to match `validate-package-metadata` pattern.
- `orchestrator/src/index.mjs`: `failedStep` was destructured but never used. Fixed with `_failedStep`.
- `validate-source-imports/src/rules.mjs`: `specifier` in `message(pkg, specifier)` was unused. Fixed with `_specifier`.

## What was deferred

- `secrets:scan` script (local) ŌĆö graceful no-op if gitleaks binary not on PATH. Hard gate runs via GitHub Action in CI only.
- Knip configuration tuning ŌĆö skeleton packages produce expected false positives; promote to hard gate after first vertical slice ships real exports.
- dependency-cruiser configuration tuning ŌĆö promote to hard gate after first vertical slice defines real module graph.

## ADR compliance

Does not replace or duplicate `validate-source-imports`. ESLint intentionally omits ADR import-boundary rules ŌĆö the custom validator is the authoritative boundary gate (ADR-0001, ADR-0002, ADR-0013, ADR-0014, ADR-0015).
