Generated by the closure pass (2026-06-17). Records the static verification + the focused

## Scope delivered

- **Scenario manifest (A)** — `e2e/scenario-manifest.json` is the canonical machine-readable
  source of correlatable scenarios with stable explicit ids (never sanitised titles). Specs
  declare ids via `test.use({ scenarioId })` / the `scenario()` annotation
  (`e2e/support/correlation.ts` + `correlation-core.mjs`). `tools/e2e/validate-scenario-manifest`
  gates: every test maps to exactly one scenario / dynamic expansion / exemption; ids globally
  unique + stable; stage policy compatible with the suite; observability fields valid; no
  title-derived-only scenario. Wired into the `e2e-coverage-validate` group.
- **Correlation coverage** — `correlatedApiContext` injects `x-e2e-*` headers on `page.request`
  / the `request` fixture / helper calls (previously bypassed), strictly same-origin (never to
  Keycloak / Cloudflare / third parties).
- **Per-scenario completeness (B)** — `observability-correlation` queries Loki per scenario with
  real backward pagination (no fixed 200-line blind spot; truncation surfaced honestly) and
  reports expected / observed / missing / unexpected. A missing REQUIRED scenario = FAILED. The
  synthetic probe is its own manifest entry (`pipeline-health-probe`) and does not substitute
  for real scenarios.
- **Real Tempo trace assertion (C)** — `tools/e2e/tempo-trace.mjs`: Loki traceId → Tempo
  `/api/traces/<id>` (polled for ingest) → OTLP parse → assert expected service/span + route +
  no leaked secrets. Result semantics: backend unreachable/unconfigured when required = DEGRADED
  (exit 2); Loki line but no trace after polling = FAILED; trace missing a required service/span
  = FAILED; all match = FULL.
- **Tempo availability (D)** — `scripts/compose/up.sh` observability profile now starts
  `loki tempo grafana alloy`; per-stage `TEMPO_HTTP_PORT` (3210–3213) already in the manifests.
- **Honest ladder (E)** — `verify-ladder` requires ALL FOUR stages FULL (degraded anywhere
  fails). Direct `make stage-<stage>` returns exit 2 on DEGRADED; only the `all-promote`
  orchestrator sets `LADDER_CONTINUE_ON_DEGRADED=1` to collect later-stage evidence; FAILED
  always halts. Decision extracted to `scripts/stages/stage-exit.sh`.
- **Evidence attestation (F)** — evidence-only-commit model: the tested commit is HEAD, or an
  ancestor whose diff to HEAD touches only `docs/evidence/**` / `docs/adr/**`. Committing the
  generated evidence no longer invalidates the ladder; changing any source/tooling path after
  the tested run does (re-run required).
